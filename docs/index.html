Adopt In 5 Minutes

Pick the path that matches how you want to use the project.

Public usage is split into two lanes. Use the GitHub Action in your own repository for CI gating, or run the CLI from a source checkout when you want local evaluation. npm publication is reserved for a later maintainer backfill release.

GitHub Action in your repo

The supported consumer path for CI usage in a separate repository.

jobs:
  web-quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2
        with:
          persist-credentials: false
      - uses: Jahrome907/web-quality-gatekeeper@v3
        with:
          url: https://your-site.example
          baseline-dir: baselines

Run the CLI from source

Use this path when you want local evaluation before wiring CI.

git clone https://github.com/Jahrome907/web-quality-gatekeeper.git
cd web-quality-gatekeeper
npm ci
npx playwright install
npm run build
node dist/cli.js audit https://your-site.example --policy marketing

Keep custom settings in a consumer-owned path such as .github/web-quality/config.json rather than copying repo-internal maintainer paths.

Workflow

One run, both review surface and automation surface.

One audit writes the review artifacts and summary contracts that teams usually assemble by hand across separate tools.

  1. Resolve the target

    Validate the URL, apply SSRF-sensitive guardrails when needed, and pin the resolved host so redirects stay deterministic.

  2. Open the page once

    Playwright loads the page and captures runtime signals; axe, Lighthouse, and visual diff run against the same resolved target.

  3. Emit stable outputs

    Every run writes report.html, summary.json, and summary.v2.json, plus optional baselines and trend artifacts.

  4. Gate releases

    The same summary contract drives PR comments, CI exit codes, and release tag movement when the workflow is configured for it.

Inspectable contracts

  • Schemas live in schemas/summary.v1.json and schemas/summary.v2.json.
  • Every published proof artifact links to its generating config and the SHA it was produced from.
  • The HTML report on this page is the same artifact wqg audit emits locally.

Operational guarantees

  • Internal/private targets are blocked by default in CI and on authenticated runs.
  • action.yml inputs and outputs remain compatible unless explicitly extended in CHANGELOG.md.
  • Release major tags only move for stable vX.Y.Z tags — never for prereleases.
